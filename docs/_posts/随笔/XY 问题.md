## 这是什么？
XY 问题指的是：提问者描述和求助的其实是自己尝试的解决方案（Y），而不是真正想要解决的实际问题（X）。<!-- more -->  
这种问法会导致提问者和回答者双方都浪费大量时间和精力。

### 典型过程如下：
- 用户真正想要实现的是 X
- 用户不知道怎么做 X，但觉得自己只要能完成 Y，就能勉强搞定 X
- 用户其实也不知道怎么做 Y
- 用户跑来问：请教怎么做 Y？
- 其他人努力帮忙解答 Y，但心里觉得：这个 Y 本身就很奇怪、目的不明确
- 经过大量来回、反复解释、浪费很多时间后，终于搞清楚：用户真正想要的是 X，而 Y 根本不是一个合适的（甚至可能是完全错误的）解决办法

**核心原因：** 提问者过于执着于自己脑补的解决方案，无法退后一步，把完整的需求和背景讲清楚。

## 应该怎么做才能避免 XY 问题？
- 提问时，尽量同时说明更大的背景和最终目标，而不仅仅抛出你当前卡住的那个具体步骤
- 如果对方问你要更多上下文、信息、完整需求，请尽量详细提供
- 如果你已经尝试过其他方法但排除了，也请说明“为什么排除”，这能让对方更快理解你的真实限制条件和偏好

**如果你对问题的理解和解决方案思路是完全正确的，你根本就不需要来问别人了，对吧？**

## 真实例子

### 示例一
新手其实并不是想要文件名最后 3 个字符，他真正想要的是文件扩展名，所以为什么要问“最后 3 个字符”呢？
```
<bob> 怎么 echo 出文件名最后三个字符？
<feline> 如果在变量里：echo ${foo: -3}
<feline> 为什么要正好三个字符？你到底真正想要什么？
<feline> 是想要文件扩展名吗？
<bob> 对，就是扩展名。
<feline> 但并不是所有文件扩展名都是三个字母啊。
<feline> 直接截最后三个字符并不能可靠解决问题。
<feline> 正确的做法是：echo ${foo##*.}
```

### 示例二
如果 Angela 一开始就直接说明她想防止别人探测出她的操作系统，这场讨论本可以变得短得多，也高效得多。
```
<Angela> nmap -O -A 127.0.0.1 会输出几行 OS: 开头的信息，怎么去掉？
<Obama> 你得去看 nmap 源码，搞明白它怎么识别 Linux 的，然后改写你自己的 TCP/IP 协议栈，让它不被 nmap 认出来。
<Angela> 可是我完全不懂 Linux 系统 API 啊。。。
<Obama> nmap 的指纹识别就是基于 TCP/IP 协议栈的行为，除了改协议栈基本没其他办法。
<Angela> 我真的很需要去掉这些信息，iptables 能做到吗？
<Obama> 那就别开 OS 检测和版本扫描啊。。。
<Angela> （终于说清楚）我就是想防止别人知道我用的是什么操作系统！
```
